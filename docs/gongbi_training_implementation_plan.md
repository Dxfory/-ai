# 工笔花鸟 AI 教练训练与落地实施方案

## 当前判断

我们不应该把核心能力简单定义成“成品图生成白描图”。真实目标是让系统学会工笔花鸟的观察和制作逻辑：

- 形理：真实物象如何被取舍、概括、装饰化，并组织进画面。
- 白描：哪些线是骨架线，哪些线表达质感、转折、遮挡、虚实和节奏。
- 设色：对象、颜色、动作、顺序、干湿条件和常见问题如何绑定。
- 教学：把一张用户上传图拆成一步一页的可执行练习。

刚刚对 `book_001` 的第 18、26、27、30 页做了视觉/OCR 抽取，结果验证了这条路线可行：

- 第 18 页：能抽取香瓜、瓜叶、螽斯、牛筋草从真实物象到画面形理的转化。
- 第 26 页：能抽取石榴、葡萄、柑橘的真实物象对照和装饰化组织方式。
- 第 27 页：能抽取白描质量规则，例如质感区分、包裹转折、笔断意连、线条节奏。
- 第 30 页：能抽取设色步骤，例如曙红勾石榴籽、汁绿罩染正叶、赭石加藤黄罩染反叶、胭脂分染葡萄、蛤粉圈线后淡胶矾水固色。

## 阶段 1：整书解析数据管线

目标：用户只提供完整书籍扫描或 zip，不要求人工先分类。

已完成：

- 整书导入：`scripts/prepare_teaching_pages.py`
- 页面联系表和清单：`pages.jsonl`
- 标注模板：`annotation_template.json`
- 粗版面候选：`scripts/extract_page_regions.py`
- 视觉/OCR 抽取：`scripts/extract_teaching_units.py`

下一步：

1. 扩展到整本 `book_001`，但先按页组跑，不一次性全跑。
2. 每 5-8 页人工抽查一次，修正 prompt 和 schema。
3. 把稳定结果汇总成一本书的 `book_knowledge.json`。

## 阶段 2：知识库结构

目标：把每本书沉淀成可检索、可复用的工笔花鸟教学知识。

建议结构：

- `artwork_cases`：一幅作品或一个题材的完整案例。
- `figure_bindings`：图号、图像内容、图注、正文解释的绑定关系。
- `form_logic_units`：真实物象到画面结构的转化规则。
- `baimiao_rules`：白描线条质量规则。
- `technique_units`：设色动作单元。
- `step_sequences`：从勾线到完成的顺序链。
- `review_flags`：需要人工确认的 OCR 或图像理解不确定项。

第一版可以用 JSON 文件，不急着上向量数据库。等资料超过 5-10 本、检索需求明确后再引入向量索引。

当前 `book_001` 已可生成第一版轻量知识文件：

```bash
.venv/bin/python scripts/build_book_knowledge.py datasets/processed_books/book_001
```

它会读取本地 `ocr/` 结果，输出 `book_knowledge.json`。这个文件只保存页级摘要、图号索引、已有墨线规则、技法单元和忠实度约束，不复制完整 OCR 长文本。

## 阶段 3：白描生成短期路线

短期不要完全依赖 `gpt-image-2` 直接重画白描，因为它会改变构图和细节。

短期方案应是三层：

1. 本地图像处理生成边缘/结构候选，只作为参考底图。
2. 视觉模型分析原图，输出对象、结构、遮挡、骨架线说明。
3. 让用户或人工修图环节基于候选底图修正，沉淀“原图 -> 人工合格白描”的对照样本。

MVP 里可以把 AI 白描标注为“参考白描”，而不是承诺完全可用。

验收标准：

- 构图方向不能变。
- 对象数量不能变。
- 主枝、主叶、花果/鸟体位置不能变。
- 允许简化纹理，但不能发明结构。
- 线稿必须黑白清晰，少灰度、少噪声。

## 阶段 4：白描生成长期路线

长期需要小模型或工作流辅助，而不是只换提示词。

推荐路径：

1. 收集 30-50 组“原图/合格白描”作为评估集。
2. 收集 100-300 组作为第一阶段微调或 LoRA 数据。
3. 对每组数据标注对象类别、是否遮挡、线条层级、人工修改点。
4. 训练或微调一个“忠实线稿提取”模型。
5. 用规则评估和人工评估共同判断：结构忠实度、线条清晰度、白描风格、可临摹性。

这里的训练目标不是让模型“画得好看”，而是让它“少改原图、会取舍、线条符合工笔逻辑”。

## 阶段 5：教学页生成

输入：用户上传图片 + 白描稿 + 教材知识库。

输出：

1. 白描稿与构图定位。
2. 勾线说明：对象分组、主线、辅线、质感线。
3. 初步分染：先处理哪类对象，颜色和水分如何控制。
4. 罩染/统染：如何统一色调，不压死线条。
5. 提染/细节：花心、叶脉、羽毛、籽粒、果皮纹理。
6. 复勾/完成检查：哪里需要醒线，哪里不能加重。

每一步都要包含：

- 当前目标。
- 参考图或局部图。
- 操作动作。
- 材料和颜色。
- 常见错误。
- 用户拍照上传。
- 叠图或局部对照。

## 阶段 6：内测数据闭环

第一轮内测不要扩大题材，继续工笔花鸟局部。

收集：

- 用户上传原图。
- AI/人工白描稿。
- 每一步用户照片。
- 用户卡住的位置。
- 用户认为看不懂的说明。
- 最终作品。
- 人工老师修正建议。

这些数据以后会反哺：

- 白描忠实度。
- 步骤说明清晰度。
- 错误提醒库。
- 适合初学者的题材筛选。
