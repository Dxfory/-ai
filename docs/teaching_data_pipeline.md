# 工笔花鸟整书训练资料准备流程

## 目标

这条数据线不要求你提前把资料分成“原图、白描、上色步骤”。相反，我们先把整本技法书当作完整页面导入，保留图片、文字、图号、页序和上下文，再逐步让系统识别其中的教学逻辑。

最终要沉淀出三类能力：

- 白描结构理解：从成品图中找出花、叶、枝、鸟的骨架关系和应保留的墨线。
- 步骤技法理解：知道勾线、分染、罩染、提染、复勾分别解决什么问题。
- 教学页面生成：面对用户上传的新图，能按工笔花鸟逻辑拆成可执行的一步一页。

## 当前导入方式

把整本扫描资料打成一个 `.zip`，里面可以是 `.jpg`、`.png`、`.webp`、`.tif` 等图片，不需要手动分类。

运行：

```bash
.venv/bin/python scripts/prepare_teaching_pages.py /path/to/book.zip --book-id book_001
```

脚本会生成：

- `datasets/raw_books/book_001/page_001.jpg...`：原始页面，本地保存，不提交 Git。
- `datasets/processed_books/book_001/pages.jsonl`：每页尺寸、顺序、文件路径等元数据。
- `datasets/processed_books/book_001/annotation_template.json`：后续 OCR、图像区域、技法单元的标注模板。
- `datasets/processed_books/book_001/contact_sheet.jpg`：快速总览图，本地保存，不提交 Git。

## 后续抽取层级

### 1. 页面层

先确认页序、页面类型和清晰度：

- 作品整图页
- 局部观察页
- 白描/线稿页
- 分步骤设色页
- 文字技法说明页
- 作者/风格说明页

### 2. 图文绑定层

每页识别：

- OCR 全文
- 文本块位置
- 图像区域位置
- 图号，例如 `图9`、`图10`
- 图注和正文中对图号的引用

这一步的关键不是只取文字，而是把“哪段文字在解释哪张图”绑定起来。

### 3. 技法单元层

从图文绑定结果里抽取可复用的技法单元：

- 形理：物象本身如何生长、转折、穿插、遮挡，画中做了哪些取舍。
- 对象：花瓣、正叶、反叶、枝干、鸟头、鸟背、羽毛、果实。
- 材料：墨、花青、赭石、藤黄、曙红、胭脂等。
- 动作：勾线、平涂、分染、罩染、提染、醒线、复勾。
- 顺序：先后关系和等待干透等条件。
- 警告：过厚、脏、糊线、结构不清、层次不透。

### 4. 训练与评估层

等积累到足够资料后，再生成两类样本：

- 训练样本：完整页面上下文 -> 图文绑定/技法单元/步骤拆分。
- 评估样本：用户上传新图 -> 白描骨架/步骤说明，需要人工检查是否忠实。

## 这份 `扫描分步训练.zip` 的初步观察

当前已导入 `book_001`，共 33 页。视觉上包含：

- 多张完整花鸟作品页。
- 多张白描或浅色线稿页。
- 多张局部观察与对照页。
- 多张分步骤设色示范页。
- 多页文字说明和图例组合页。

它适合作为第一本“整书理解”样本，用来验证 OCR、图号绑定、技法单元抽取，而不是只拿来训练原图到白描的图片对。

已细看样页：

- 第 18 页和第 26 页偏“形理说明”：作者把画中物象和真实照片并置，解释形态、取舍、枝叶/果实/昆虫的结构关系。
- 第 27 页偏“白描逻辑”：强调勾线要区分物象质感、线条转折、包裹关系和线与线之间的虚实衔接。
- 第 30 页偏“设色步骤”：把图号、局部图和动作绑定到一起，例如不同对象使用不同罩染、分染、圈线、固色处理。

## 下一步

下一步应该做 OCR 和图文绑定原型：

1. 对 `page_001...page_033` 做 OCR。
2. 自动检测页面里的图像区域和文本区域。
3. 识别 `图x` 编号，并把相邻正文绑定到对应图像。
4. 先抽 3-5 页人工复核，确认“技法动作、颜色、对象、顺序”抽取是否符合你的书里逻辑。
5. 通过人工复核样本再扩展到更多书。

## 页面区域候选

当前已有一个轻量版页面分析脚本：

```bash
.venv/bin/python scripts/extract_page_regions.py datasets/processed_books/book_001 --pages 18,26,27,30 --write-previews
```

它会生成：

- `layout_candidates.json`：文字区、图像区、图号/图注候选框。
- `layout_previews/`：带框预览图，本地检查用，不提交 Git。

这个脚本只做“粗区域候选”。它的价值是给 OCR、图号绑定和人工复核提供入口，不是最终的自动理解模型。对于书脊阴影、旁页残影、浅色画面底纹，它可能会把多个图块合并成大块；后续接入更强版面模型后再细分。

## 视觉/OCR 抽取

如果 `.env` 中配置了 OpenAI 兼容视觉模型，可以用下面的脚本抽取结构化教学信息：

```bash
.venv/bin/python scripts/extract_teaching_units.py datasets/processed_books/book_001 --pages 18,26,27,30 --dry-run
.venv/bin/python scripts/extract_teaching_units.py datasets/processed_books/book_001 --pages 18,26,27,30 --max-image-side 1600
```

建议配置：

```bash
TEACHING_VISION_API_BASE=https://lingsuan.top
TEACHING_VISION_API_KEY=replace-with-your-api-key
TEACHING_VISION_MODEL=gpt-5.5
```

脚本输出到 `datasets/processed_books/book_001/ocr/`。这个目录不会提交 Git，因为里面可能包含书籍 OCR 文本和模型抽取结果。第一轮只跑少量页，人工确认抽取结果靠谱后，再扩展到整本书。

默认会先把扫描页压缩到最长边 1600px，再发给视觉模型，避免整页原图过大导致请求过慢或失败。
