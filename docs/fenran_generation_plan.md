# 工笔花鸟分染生成规划

## 当前结论

白描只是第一步。接下来系统要学习的是：白描线稿搭好骨架之后，如何用分染建立对象体积、正反叶关系、枝干质感、果实前后层次和鸟体明暗。

分染图不是完整成品，也不是给线稿平涂颜色。它应该像教材步骤图：线还清楚，颜色很淡，局部过渡自然，能让用户知道下一步从哪里开始、颜色多重、关系如何建立。

## 已从 33 页教材抽到的分染规律

- 正叶：花青、汁绿、三绿体系，顺主叶脉和叶面转折分染，主叶脉附近保留水线。
- 反叶：赭石加少量藤黄、四绿或汁绿，色相更灰、更柔，和正叶区分开。
- 枝干：淡墨、赭墨、赭石加藤黄，强调干笔皴染、节疤、转折和遮挡。
- 果实：赭粉、赭石、藤黄、淡胭红、胭脂，按圆润体积和前后遮挡少量多次分染。
- 鸟体：淡墨、赭墨、赭石，按头、身、尾、胸腹和羽组关系分染，不新增羽毛。
- 花和白粉：花瓣转折要轻，白粉/蛤粉只用于醒提花脉或亮部，不能变成装饰纹样。

## MVP 生成链路

1. 用户上传原画。
2. 系统生成白描稿。
3. 用户点击生成分染图。
4. 后端把原画和白描稿一起交给图片模型：
   - 原画负责颜色、对象和完成效果参考。
   - 白描稿负责锁定线稿、构图和结构。
5. 模型输出淡彩分染步骤图。

当前新增接口：

```text
POST /api/v1/color-steps/generate
```

请求字段：

```json
{
  "reference_upload_id": "原画上传 ID",
  "line_draft_id": "白描稿 ID",
  "step_type": "fenran",
  "provider": "ai_fenran"
}
```

## 质量标准

分染图必须满足：

- 构图方向不变。
- 对象数量不变。
- 白描线清楚保留。
- 颜色透明、过渡自然。
- 正叶、反叶、枝干、果实、鸟体的处理逻辑不同。
- 不生成完整浓彩成品。
- 不新增枝叶、花瓣、羽毛、果实或装饰纹样。

## 短期实现

短期继续使用现有 LingSuan 图片接口，重点优化提示词和评估样本。

需要我们做：

- 用 3-5 张用户可授权的原画测试白描 + 分染链路。
- 每张保存：原画、AI 白描、AI 分染、人工评价、失败原因。
- 失败原因按类型记录：
  - 构图改变。
  - 线被盖住。
  - 颜色太重。
  - 正反叶不分。
  - 过渡生硬。
  - 生成了最终成品。
  - 新增了不存在的结构。

## 长期实现

等素材更多后，建立真正的步骤图训练集。

推荐最小数据量：

- 10-20 组：只够做提示词和人工评估集。
- 30-50 组：可以稳定比较不同模型/参数。
- 100-300 组：可以开始考虑 LoRA、ControlNet 或专门的图像转换小模型。
- 500+ 组：才更适合训练较稳定的“原画/白描 -> 分染步骤图”能力。

每组最好包含：

- 原画或成品图。
- 白描/勾线图。
- 分染步骤图。
- 后续罩染水色、罩染石色、提染、复勾或完成稿。
- 对应教材文字说明。
- 人工标注：对象、颜色、动作、顺序、注意事项。

## 下一阶段

白描和分染跑通后，继续把步骤扩展成：

1. 白描。
2. 分染。
3. 罩染水色。
4. 罩染石色。
5. 提染/醒提。
6. 复勾与完成检查。

每新增一步都先做同样的规则文件、生成接口和页面按钮，再进入测试。
